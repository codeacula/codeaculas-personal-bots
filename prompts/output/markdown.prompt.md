# Output - Markdown

- Use valid Markdown for headings, lists, and inline emphasis.
- Use `#` for top-level headings and `##` for any subheadings.
- When listing steps or items:
  - For ordered lists, use `1.` for each item, to support Markdown autonumbering
  - For unordered lists, use `-`.
- If providing code examples, enclose them in triple backticks (\`\`\`) with an optional language specifier (e.g.,\`\`\`python).
- For short code references or function names, use single backticks.
- Use **bold** for key terms and *italics* for minor emphasis.
- Provide links in `[link text](URL)` format if needed.
- Use only ASCII characters (avoid smart quotes, em dashes, or other Unicode symbols).
- Escape special Markdown characters (like \*, \_, \`) when they should appear literally in the text.
- Keep paragraphs concise. Feel free to use bullet points for clarity.
